# CLAUDE.md — minimal-react-harness

> Claude Code가 세션 시작 시 자동으로 읽는 프로젝트 컨텍스트. 한 번 읽고 모든 작업에 적용한다.

## 프로젝트 개요
- 작은 언어 모델(sLM)을 두뇌로 쓰는 **ReAct 에이전트 하네스(harness)**를 프레임워크 없이 직접 구현한 학습/포트폴리오 레포.
- 배경: 삼성전자 **SCPC 2026 AI 챌린지**(주제: AI Agent Harness 설계·구현, sLM) 대비 + 포트폴리오.
- 목표: ReAct 루프(생각 → 도구 → 관찰 반복)의 동작 원리를 코드로 명확히 드러내고, 작은 모델의 형식 오류를 하네스가 방어함을 보인다.

## 핵심 제약 (반드시 지킬 것)
- **추론 경로에 외부 API 호출 금지**(OpenAI/Gemini 등). 모델은 로컬에서 직접 실행(load)한다.
  - ※ 이는 2025 규칙 기준. 2026 공식 규칙(파라미터 한도·모델 공개일·라이선스)은 7/6경 공개 예정 — 확정 전까지 "작은 로컬 모델" 가정을 유지한다.
- 사용 언어: Python.
- **재현성**: 같은 입력 → 같은 출력이 되도록 그리디 디코딩 등 결정적 설정 유지.
- 모델 추가/교체 시 license·공개일이 대회 규칙에 맞는지 model card로 확인.

## 아키텍처
- `react_agent.py` : 하네스 엔진(파싱·도구 실행·Observation 주입·가드레일). 모델은 생성자에 주입.
- `tools.py`       : 도구 함수(`search`=로컬 KB 스텁, `calculate`=안전 계산). 새 도구는 `TOOLS`에 등록.
- `llm.py`         : 백엔드 — `MockLLM`(테스트), `HFLocalLLM`(로컬 모델). 규약: `(prompt, stop) -> str`, 'Thought:'로 시작.
- `main.py`        : 실제 모델 실행 진입점.
- `test_harness.py`: `MockLLM`으로 엔진 검증(GPU 불필요, 항상 통과 유지).

## 실행
- 엔진 검증: `python test_harness.py`  (의존성/GPU 불필요)
- 실제 모델: `pip install -r requirements.txt` 후 `python main.py`  (GPU 권장)

## 작업 규칙
- 답변·설명은 **한국어**로. 기술 용어(영문)는 번역하지 말고 그대로 둔다.
- 코드 변경 **전에 계획부터 제시**하고, 승인받은 뒤 실행한다.
- 기존 파일을 크게 바꾸거나 삭제하기 전엔 먼저 물어본다.
- 변경 후에는 `python test_harness.py` 통과를 확인한다.
- 이 레포는 학습/포폴용 — 핵심 동작에는 **의도를 설명하는 간결한 주석을 유지**한다(과한 주석은 지양).
- 추론 경로에 외부 API를 넣지 않는다. `search`는 로컬 리트리버로만 확장한다.

## 로드맵 (다음 작업 후보)
1. `main.py`에 실제 작은 모델을 연결해 end-to-end 1회 확인 + 형식 준수 관찰.
2. 작은 모델의 Action 형식 오류 대응 강화(파서 견고화, 재시도 한도, JSON 스키마 옵션).
3. `search`를 실제 로컬 리트리버(위키 덤프/임베딩 검색 등)로 교체.
4. 트레이스 로그를 `examples/`에 파일로 저장 — 포폴 캡처용.
5. 여러 질문에 대한 정답률을 재는 간단한 평가 스크립트 추가.
